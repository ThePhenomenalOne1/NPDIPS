================================================================================
KNOWN SCHEMA ISSUES
================================================================================

  1. DUAL COLLECTION CONFLICT
     "stores" (used by app) vs "shops" (used by seed service + security rules).
     Security rules for "shops" do NOT protect documents in "stores".
     Recommendation: Migrate rules and seed service to use "stores".

  2. HARDCODED TWILIO CREDENTIALS
     twilio_otp_service.dart and functions/otp.js embed live Twilio credentials
     in source code. These should be rotated and moved to environment variables.

  3. OTP SESSION SECURITY
     otp_sessions collection has open read/write rules (no auth required).
     Recommendation: Restrict to authenticated users or remove if unused.

  4. OTP STORED IN PLAINTEXT
     otp_sessions stores the raw OTP code in Firestore.
     Recommendation: Store only a bcrypt/HMAC hash instead.
